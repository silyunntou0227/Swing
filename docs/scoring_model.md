## 1. アーキテクチャ概要

```
データ取得 → フィルタリング(4層) → テクニカル指標計算 → 11因子スコアリング → マクロ調整 → ランキング → 通知(Discord/LINE)
```

- 全個別スコアは **0〜100** に正規化（基準値50）
- 加重平均でベーススコアを算出し、マクロ調整（±5pt）を加算
- 最終スコア = `max(0, min(100, base_score + macro_adjustment))`

## 2. 総合スコア算出

### 因子ウェイト

| 因子 | ウェイト | 理論根拠 |
|------|----------|----------|
| トレンド | 0.20 | トレンドフォロー戦略の主軸 |
| MACD | 0.14 | モメンタム転換の検出 |
| 一目均衡表 | 0.12 | 日本市場特化の複合トレンド指標 |
| 出来高 | 0.11 | 価格変動の裏付け確認 |
| ファンダメンタル | 0.10 | 割安・成長性のフィルタ |
| RSI | 0.10 | 過熱/売られすぎの検出 |
| パターン | 0.05 | ローソク足形状による反転シグナル |
| リスクリワード | 0.05 | ボラティリティ適正度 |
| ニュース・開示 | 0.05 | 材料による追加バイアス |
| セクター | 0.05 | 相場サイクルとの整合性 |
| 信用残 | 0.03 | 需給バランス |
| **合計** | **1.00** | |

### 計算式

```
base_score = Σ (individual_score[i] × weight[i])

total_score = max(0, min(100, base_score + macro_adjustment))
```

> `macro_adjustment` は −5 〜 +5 の範囲（日経平均のモメンタムに基づく）

## 3. 個別スコアリングモジュール

### 3.1 トレンドスコア (Weight: 0.20)

**理論**: 移動平均線のパーフェクトオーダー（5日 > 25日 > 75日 > 200日）はトレンドの強さを示す。

**SMAパラメータ**:
- 短期: 5日 / 中期: 25日 / 長期: 75日 / 超長期: 200日

**採点ロジック** (基準値50):

| 条件（買い方向） | 加点 |
|------------------|------|
| 終値 > SMA5 > SMA25 | +25 |
| + SMA25 > SMA75（パーフェクトオーダー） | +15 |
| 終値 > SMA5 のみ | +10 |

売り方向は符号を反転して適用。

---

### 3.2 MACDスコア (Weight: 0.14)

**理論**: Kang (2021) による日本株最適パラメータ。標準の (12,26,9) ではなく、東証の短期サイクルに合わせた (4,22,3) を採用。

**パラメータ**:

| 要素 | 標準値 | 本システム | 理由 |
|------|--------|------------|------|
| 短期EMA | 12 | **4** | 日本市場の高速サイクル対応 |
| 長期EMA | 26 | **22** | 日経225最適化 |
| シグナル | 9 | **3** | ダマシ低減 |

**計算式**:
```
MACD Line   = EMA(4) − EMA(22)
Signal Line = EMA(MACD Line, 3)
Histogram   = MACD Line − Signal Line
```

**採点ロジック** (基準値50, 買い方向):

| 条件 | 加点 |
|------|------|
| MACD > Signal | +20 |
| Histogram > 0 | +10 |
| Histogram が前日より拡大 | +10 |

---

### 3.3 出来高スコア (Weight: 0.11)

**理論**: 出来高は価格変動の確信度を裏付ける。OBV (On Balance Volume) はGranvilleが提唱した累積出来高指標。

**出来高比率**:
```
Volume Ratio = 当日出来高 / 20日平均出来高
```

**OBV計算式**:
```
if Close[t] > Close[t-1]: OBV[t] = OBV[t-1] + Volume[t]
if Close[t] < Close[t-1]: OBV[t] = OBV[t-1] - Volume[t]
if Close[t] = Close[t-1]: OBV[t] = OBV[t-1]
```

**採点ロジック** (基準値50):

| 条件 | 加点 |
|------|------|
| 出来高比率 > 2.0 | +30 |
| 出来高比率 > 1.5 | +20 |
| 出来高比率 > 1.2 | +10 |
| OBV 10日間単調増加 | +10 |
| OBV 10日間上昇傾向 | +5 |

---

### 3.4 RSIスコア (Weight: 0.10)

**理論**: デュアルRSI戦略。RSI(14) で中期トレンドを確認し、Connors (2009) のRSI(2) で短期平均回帰のエントリータイミングを計測。

**パラメータ**:

| 指標 | 期間 | 用途 |
|------|------|------|
| RSI(14) | 14日 | 中期トレンド確認 |
| RSI(2) | 2日 | 短期平均回帰（Connors RSI） |

**RSI(14) 採点** (買い方向):

| 条件 | 加点 |
|------|------|
| 30 ≤ RSI ≤ 45（売られすぎからの回復） | +12 |
| 45 < RSI ≤ 60（中立〜やや強い） | +6 |
| RSI > 70（過熱警告） | −8 |

**RSI(2) 採点** (買い方向):

| 条件 | 加点 | 解釈 |
|------|------|------|
| RSI(2) < 5 | +25 | 極端な売られすぎ＝最強シグナル |
| RSI(2) < 10 | +18 | 強い売られすぎ |
| RSI(2) < 20 | +8 | 軽度の売られすぎ |
| RSI(2) > 90 | −10 | 過熱 |

> RSI(2) に60%、RSI(14) に40% のウェイト配分

---

### 3.5 一目均衡表スコア (Weight: 0.12)

**理論**: 一目山人が開発した日本発の複合トレンド分析。時間論・波動論・値幅観測論の三理論に基づく。

**パラメータ**: 転換線9日 / 基準線26日 / 先行スパンB 52日

**計算式**:
```
転換線     = (9日間最高値 + 9日間最安値) / 2
基準線     = (26日間最高値 + 26日間最安値) / 2
先行スパンA = (転換線 + 基準線) / 2    → 26日先にプロット
先行スパンB = (52日間最高値 + 52日間最安値) / 2  → 26日先にプロット
遅行スパン  = 終値を26日前にプロット
雲         = 先行スパンA 〜 先行スパンB の領域
```

**採点ロジック** (基準値50, 買い方向):

| 条件 | 加点 |
|------|------|
| 転換線 > 基準線 | +10 |
| 終値 > 雲上限（雲の上に位置） | +20 |
| 終値が雲の中 | +5 |

**特殊シグナル**:
- **三役好転**: 転換線>基準線 + 終値>雲 + 遅行スパン>26日前終値 → 最強の買いシグナル
- **三役逆転**: 上記の逆条件 → 最強の売りシグナル

### 3.6 ファンダメンタルスコア (Weight: 0.10)

**理論**: バリュー投資のスクリーニング基準。割安かつ収益性の高い銘柄を選好。

**PER (株価収益率) 採点** (基準値50):

| 条件 | 加点 | 解釈 |
|------|------|------|
| 8 ≤ PER ≤ 15 | +15 | 割安 |
| 15 < PER ≤ 25 | +5 | 適正 |
| PER > 35 | −10 | 割高 |

**PBR (株価純資産倍率) 採点**:

| 条件 | 加点 | 解釈 |
|------|------|------|
| 0.5 ≤ PBR ≤ 1.0 | +15 | 割安 |
| 1.0 < PBR ≤ 2.0 | +5 | 適正 |
| PBR > 3.0 | −5 | 割高 |

**ROE (自己資本利益率) 採点**:

| 条件 | 加点 |
|------|------|
| ROE ≥ 15% | +15 |
| ROE ≥ 10% | +10 |
| ROE ≥ 5% | +5 |

### 3.7 ローソク足パターンスコア (Weight: 0.05)

**理論**: 酒田五法を基礎としたローソク足パターン認識。

**検出パターン**:

| パターン | 検出条件 | 方向 |
|----------|----------|------|
| 包み足（強気） | 前日陰線 + 当日陽線が前日実体を完全包含 | 買い |
| 包み足（弱気） | 前日陽線 + 当日陰線が前日実体を完全包含 | 売り |
| はらみ足（強気） | 前日陰線 + 当日陽線が前日実体の内側 | 買い |
| はらみ足（弱気） | 前日陽線 + 当日陰線が前日実体の内側 | 売り |
| 十字線 | 実体 < 全値幅の10% | 転換 |
| ハンマー | 下ヒゲ ≥ 実体の2倍 かつ 上ヒゲ ≤ 実体の0.3倍 | 買い |
| 流れ星 | 上ヒゲ ≥ 実体の2倍 かつ 下ヒゲ ≤ 実体の0.3倍 | 売り |
| 赤三兵 | 3本連続陽線 + 各終値が前日終値を上回る | 買い |
| 三羽烏 | 3本連続陰線 + 各終値が前日終値を下回る | 売り |

**採点**: 方向が一致するパターン1つにつき **+15**（基準値50）

### 3.8 リスクリワードスコア (Weight: 0.05)

**理論**: ATR (Average True Range) で測定したボラティリティが、スイングトレードに適切な範囲にあるかを評価。

**ATR計算式**:
```
True Range[t] = max(
    High[t] - Low[t],
    |High[t] - Close[t-1]|,
    |Low[t] - Close[t-1]|
)
ATR = 14日間の True Range 平均
```

**採点ロジック** (基準値50):

| 条件 | 加点 | 解釈 |
|------|------|------|
| 0.01 ≤ ATR/終値 ≤ 0.04 | +20 | 最適なボラティリティ |
| ATR/終値 > 0.06 | −10 | 過剰なボラティリティ |

---

### 3.9 ニュース・開示スコア (Weight: 0.05)

**理論**: 適時開示・ニュースセンチメントによる材料バイアス。

**TDnet適時開示スコア**:

| イベント | スコア |
|----------|--------|
| 業績上方修正 | +12 |
| 増配 | +8 |
| 自社株買い | +8 |
| 株式分割 | +5 |
| 業績下方修正 | −18 |
| 減配 | −10 |
| MBO / TOB | −999（除外） |

**EDINET大量保有スコア**:

| イベント | スコア |
|----------|--------|
| 大量保有報告書（新規） | +5 |
| 大量保有報告書（増加） | +3 |
| 大量保有報告書（減少） | −3 |

**ニュースセンチメント計算**:
```
sentiment_score = (positive_count − negative_count) / max(記事数, 1) × 5.0
sentiment_score = max(−10, min(10, sentiment_score))

total_sentiment = 個別銘柄 × 0.7 + 市場全体 × 0.3

news_score = min(100, max(0, 50 + 開示スコア + sentiment_score × 3))
```

**ポジティブキーワード例**: 増益, 最高益, 上方修正, 好調, 受注増, 黒字転換, 増配, シェア拡大 ...

**ネガティブキーワード例**: 減益, 下方修正, 不正, リコール, 訴訟, 赤字, 減配, 無配, 倒産 ...

**判定ラベル**:
- score > 3 → ポジティブ / score < −3 → ネガティブ / それ以外 → 中立

### 3.10 信用残スコア (Weight: 0.03)

**理論**: 信用倍率（買い残 / 売り残）は需給バランスを示す。売り残が多い銘柄は踏み上げ（ショートスクイーズ）の可能性がある。

**採点ロジック** (基準値50, 買い方向):

| 条件 | 加点 | 解釈 |
|------|------|------|
| 信用倍率 < 1.0 | +25 | 売り残優位 → 踏み上げ期待 |
| 信用倍率 < 2.0 | +15 | やや売り残多い |
| 信用倍率 > 5.0 | −15 | 買い残過多 → 上値重い |

### 3.11 セクターローテーションスコア (Weight: 0.05)

**理論**: 相場サイクル理論に基づくセクターローテーション分析。市場の4フェーズに応じて有利/不利なセクターが変化する。

**相場サイクル 4フェーズ**:

| フェーズ | 名称 | 特徴 |
|----------|------|------|
| 金融相場 | KINYU | 景気回復初期、低金利、金融緩和 |
| 業績相場 | GYOSEKI | 景気拡大、企業業績好調、金利上昇 |
| 逆金融相場 | GYAKU_KINYU | 金融引締、景気減速 |
| 逆業績相場 | GYAKU_GYOSEKI | 景気後退、業績悪化 |

**フェーズ判定アルゴリズム**:
```
if 市場トレンド=上昇 and 20日変化率 > 3.0%  → 業績相場
if 市場トレンド=上昇 and 20日変化率 ≤ 3.0%  → 金融相場
if 市場トレンド=下降 and 5日変化率 < -2.0%   → 逆業績相場
if 市場トレンド=下降                          → 逆金融相場
if 5日変化率 > 1.0%                           → 金融相場
if 5日変化率 < -1.0%                          → 逆金融相場
default                                       → 金融相場
```

**セクタープロファイル例** (TOPIX-17ベース):

| セクター | 分類 | 金利感応度 | 円安感応度 | ベータ | 有利なフェーズ |
|----------|------|-----------|-----------|--------|---------------|
| 銀行 | FINANCIAL | +0.9 | +0.1 | 1.1 | 逆金融, 業績 |
| 食品 | DEFENSIVE | 0.0 | −0.4 | 0.7 | 逆業績 |
| 電気・精密 | GROWTH | −0.4 | +0.5 | 1.2 | 金融, 業績 |

**採点計算** (基準値50):
```
adjustment = 0

# フェーズ整合性
if セクターの有利フェーズに現在フェーズが含まれる:
    adjustment += 5.0
else:
    if DEFENSIVE: adjustment -= 1.0
    else:         adjustment -= 3.0

# 逆業績相場でのディフェンシブ優遇
if フェーズ == 逆業績相場:
    if DEFENSIVE: adjustment += 3.0
    if CYCLICAL:  adjustment -= 3.0

# 業績相場でのシクリカル優遇
if フェーズ == 業績相場:
    if CYCLICAL: adjustment += 2.0

# 金融相場でのグロース優遇
if フェーズ == 金融相場:
    if GROWTH: adjustment += 3.0

adjustment = max(-10.0, min(10.0, adjustment))
sector_score = max(0, min(100, 50 + adjustment × 5))
```

## 4. マクロ調整

**理論**: 日経225のモメンタムにより市場全体の方向感を調整。個別銘柄スコアに ±5pt の加算。

**計算式**:
```
score = 0.0

# 市場トレンド
if trend = "bullish":  score += 2.0
if trend = "bearish":  score -= 2.0

# 短期モメンタム（5日変化率）
if change_5d >  2.0%: score += 1.5
if change_5d >  0.5%: score += 0.5
if change_5d < -2.0%: score -= 1.5
if change_5d < -0.5%: score -= 0.5

# 中期モメンタム（20日変化率）
if change_20d >  5.0%: score += 1.5
if change_20d >  1.0%: score += 0.5
if change_20d < -5.0%: score -= 1.5
if change_20d < -1.0%: score -= 0.5

macro_adjustment = max(-5.0, min(5.0, score))
```

## 5. フィルタリング層

スコアリング前に4層のフィルタで対象銘柄を絞り込む。

### Layer 0: ユニバースフィルタ

| パラメータ | 値 |
|-----------|-----|
| 対象市場 | プライム, スタンダード, グロース |
| 除外カテゴリ | ETF, REIT, ETN, インフラファンド |
| 最低上場日数 | 30日（IPO直後を除外） |

### Layer 1: ファンダメンタルフィルタ

| 指標 | 条件 |
|------|------|
| PER | 5.0 ≤ PER ≤ 40.0 |
| PBR | 0.3 ≤ PBR ≤ 5.0 |
| ROE | ≥ 5.0% |
| EPS成長率 | ≥ −10.0%（YoY） |
| 信用倍率 | ≤ 5.0 |

### Layer 2: トレンド & 流動性フィルタ

| 指標 | 条件 |
|------|------|
| ADX | ≥ 25（強いトレンド） |
| 平均出来高 | ≥ 50,000 株/日 |
| 平均売買代金 | ≥ 5,000万円/日 |

### Layer 3: エントリーシグナルフィルタ

| 指標 | 条件 |
|------|------|
| シグナル検出期間 | 直近3日以内 |
| 出来高スパイク閾値 | ≥ 1.5倍 |

### Layer 4: ニュース・開示除外

MBO / TOB の開示がある銘柄は自動除外（スコア = −999）。

## 6. リスク管理モデル

### 6.1 ポジションサイジング（1%リスクルール）

```
リスク金額    = 運用資金 × 0.01
リスク幅      = |ストップロス価格 − エントリー価格|
株数          = リスク金額 / リスク幅
株数          = max(100, 100株単位に切り下げ)
ポジション額  = 株数 × エントリー価格
```

| パラメータ | 値 |
|-----------|-----|
| 1トレードあたりリスク | 1.0% |
| デフォルト運用資金 | 1,000,000円 |

### 6.2 ストップロス & テイクプロフィット

**ATRベースの損益目標**:

```
【買いの場合】
ストップロス   = 終値 − ATR × 2.0
テイクプロフィット = 終値 + ATR × 2.0 × R:R比率

【売りの場合】
ストップロス   = 終値 + ATR × 2.0
テイクプロフィット = 終値 − ATR × 2.0 × R:R比率

R:R比率 = |テイクプロフィット − 終値| / |終値 − ストップロス|
```

| パラメータ | 値 |
|-----------|-----|
| ストップロス ATR倍率 | 2.0 |
| テイクプロフィット R:R比率 | 2.0（最低2:1） |

### 6.3 バルサラの破産確率

**理論**: Balsara の破産確率モデルで、現在のトレード条件で資金が枯渇する確率を推定。

**計算式**:
```
p = 勝率 (0〜1)
q = 1 − p (敗率)
R = ペイオフレシオ (平均利益 / 平均損失)
f = 1トレードあたりリスク (0〜1)

期待値 (Edge) = p × R − q
if Edge ≤ 0: 破産確率 = 100%（確実に破産）

base = q / (p × R)
exponent = 1 / f

P(破産) = base ^ exponent × 100
```

**デフォルトパラメータ**:
| パラメータ | 値 | 備考 |
|-----------|-----|------|
| 勝率 | 0.45 (45%) | 保守的な想定 |
| ペイオフレシオ | 2.0 | 実績R:Rがあればそちらを使用 |
| リスク/トレード | 0.01 (1%) | |

**解釈基準**:

| 破産確率 | 評価 |
|----------|------|
| < 5% | 優秀 |
| 5〜25% | 良好 |
| 25〜50% | 許容範囲 |
| > 50% | 危険 |

```
利益目標（残り決済） = 終値 + ATR × 3.0
```
